## 3. 指令解析技术方案

### 3.1 解析流程

```
用户语音 → Web Speech API 识别 → 文本标准化 → 正则匹配 → 参数提取 → 命令执行
```

### 3.2 支持的指令模式

```javascript
// 启动/停止
START: /^(开始|启动|开始绘图|开始画画|画画)/i
STOP: /^(停止|结束|关闭|暂停)/i

// 画布操作
CLEAR: /^(清空|清除|重来|重置|清理)/i
UNDO: /^(撤销|回退|后退|取消上一)/i
REDO: /^(重做|恢复|前进)/i

// 颜色切换
COLOR: /^(用|使用|换成|切换|选择)(.+)/i

// 尺寸调整
SIZE: /^(大|小|粗|细)(一点|一些)?/i

// 线条类型
STROKE: /^(实线|虚线|点线)/i

// 图形绘制
SHAPE: /^(画|绘制|创建)(一个?|条|个)?(.+)/i
```

## 4. 已实现功能详情

### 4.1 核心功能

| 功能 | 描述 | 状态 |
|------|------|------|
| 语音监听 | 通过"开始监听"按钮或语音指令启动 | ✅ |
| 语音识别 | 使用 Web Speech API 进行中文语音识别 | ✅ |
| 图形绘制 | 支持圆、矩形、直线、三角形、椭圆、星星、五边形、心形、十字 | ✅ |
| 颜色切换 | 支持12种基本颜色 | ✅ |
| 线条属性 | 支持粗细调整、实线/虚线切换 | ✅ |
| 撤销操作 | 撤销上一个绘制的图形 | ✅ |
| 错误处理 | 语音识别失败时自动提示和建议 | ✅ |
| 指令建议 | 无法识别时提供相似指令建议 | ✅ |
| 翻译功能 | 本地模式下将中文指令翻译为英文提示词 | ✅ |
| 关键词提取 | 从语音指令中提取动词、图形、颜色等关键词 | ✅ |
| AI 绘图 | 支持调用外部 AI API 生成图片（用户自定义配置） | ✅ |

### 4.2 错误处理机制

| 错误类型 | 错误消息 | 建议操作 | 处理方式 |
|---------|---------|---------|---------|
| no-speech | 没有检测到语音，请再说一次 | 请对准麦克风说话 | 静默处理，不打扰用户 |
| audio-capture | 无法访问麦克风 | 请检查麦克风是否连接 | 显示错误提示+语音反馈 |
| network | 网络连接失败 | 请检查网络后重试 | 显示错误提示+语音反馈 |
| not-allowed | 麦克风权限被拒绝 | 请在浏览器设置中允许麦克风权限 | 显示错误提示+语音反馈 |
| aborted | 语音识别被中断 | 点击开始监听继续 | 自动重试机制 |
| service-not-allowed | 语音服务不可用 | 请使用Chrome浏览器 | 显示错误提示+语音反馈 |
| unknown | 发生未知错误 | 请重启浏览器后重试 | 显示错误提示+语音反馈 |

### 4.3 指令识别容错

| 场景 | 处理方式 |
|------|---------|
| 无法识别指令 | 提供相似指令建议，如"是否想说: 画一个圆、画矩形?" |
| 识别置信度低 | 通过模糊匹配尝试找到最接近的指令 |
| 空白语音 | no-speech 错误静默处理，不打扰用户 |

### 4.2 备用交互方式

虽然应用主打语音控制，但为了提升可用性，提供了以下备用交互方式：

| 功能 | 描述 | 状态 |
|------|------|------|
| 按钮绘图 | 点击按钮绘制对应图形 | ✅ |
| 颜色选择器 | 点击色块快速切换颜色 | ✅ |
| 开始/停止按钮 | 控制语音监听状态 | ✅ |

## 5. 未完成功能及原因

### 5.1 重做功能 (REDO)

**计划：** 支持恢复被撤销的操作
**现状：** 仅输出提示"重做功能暂未实现"
**原因：** 历史记录系统设计较复杂，需要维护完整的操作历史栈

### 5.2 位置控制

**计划：** 支持"在左边"、"往右"等定位指令
**现状：** 图形随机分布在画布上
**原因：** 实现精确定位需要：
- 图形选中机制
- 坐标变换系统
- 相对位置解析逻辑
- 优先级低于基础绘图功能

### 5.3 尺寸控制

**计划：** 支持"大一点"、"放大"等尺寸调整
**现状：** "大一点/小一点"仅影响线条粗细
**原因：** 需要选中机制和图形变换系统

### 5.4 透明度控制

**计划：** 支持"半透明"、"不透明"指令
**现状：** 不支持
**原因：** Canvas 透明度处理较为复杂，且影响整体渲染性能

### 5.5 复合指令

**计划：** "画一个大的红色圆圈"自动拆解为多个操作
**现状：** 颜色可以正确识别，尺寸参数未处理
**原因：** 复合指令需要：
- 参数提取的优先级排序
- 上下文感知能力
- 更复杂的自然语言理解

### 5.6 椭圆绘制

**计划：** 支持"画椭圆"指令
**现状：** 不支持
**原因：** 优先级较低，可在后续版本添加

## 6. 技术限制

### 6.1 浏览器兼容性

| 浏览器 | 支持情况 | 说明 |
|--------|---------|------|
| Chrome 90+ | ✅ 完全支持 | Web Speech API 完整支持 |
| Edge 90+ | ✅ 完全支持 | 基于 Chromium |
| Safari | ⚠️ 部分支持 | 语音识别可能受限 |
| Firefox | ⚠️ 部分支持 | 语音识别可能受限 |
| 其他浏览器 | ❌ 不支持 | Web Speech API 不可用 |

### 6.2 语音识别限制

- **网络要求：** 需要网络连接（Chrome）
- **语言支持：** 主要支持中文（zh-CN）
- **环境噪音：** 可能受环境噪音影响识别准确率
- **说话速度：** 语速过快可能影响识别
